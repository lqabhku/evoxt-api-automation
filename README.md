## Evoxt API 是什么，能做什么

Evoxt 官方提供了完整的 REST API 文档，地址在 `api.evoxt.com`（需要 JavaScript 环境才能加载），本质上是一套标准的 HTTP 接口集合，让你通过代码而不是网页点击来操控你的虚拟机。

根据官网说明和第三方测评，Evoxt API 支持的核心操作包括：

- **VM 状态管理**：开机、关机、重启，全部可以通过 API 调用完成
- **重装系统**：无需登录控制台，一个请求搞定
- **防火墙配置**：Layer 3 级别的防火墙规则，可以在不连接 VM 的情况下直接设置
- **IP 地址管理**：在服务器之间交换 IP、管理浮动 IP
- **备份操作**：触发备份恢复流程
- **密码与主机名变更**：远程修改凭据，不用 VNC
- **VNC 访问**：通过 API 获取 VNC 连接信息
- **子账户管理**：给团队不同成员分配对应权限

另外值得一提的是，Evoxt 还有一个官方的 WHMCS 插件，走的也是同一套 API。也就是说，如果你想自建一个 VPS 转售业务，用 Evoxt 的 API 可以实现从部署到暂停、从续费到重装的全流程自动化，不需要自己写底层逻辑。

## 如何获取 Evoxt API 凭据

这一步很简单，不需要任何特别的申请流程：

1. 登录 Evoxt 控制台（`console.evoxt.com`）
2. 进入顶部导航或账户菜单，找到 **"API 凭据"（API Credentials）** 选项
3. 生成你的 API Key
4. 把这个 Key 保存好——它就是你后续所有 API 请求的身份凭证

Evoxt 知识库里明确列出了"API 凭据"作为账户管理的一个独立模块，说明这个功能面向所有用户开放，不需要额外付费或申请权限。

获得 API Key 之后，按照标准的 Bearer Token 方式，把它放进 HTTP 请求的 `Authorization` 头部就可以开始调用了：


Authorization: Bearer YOUR_API_KEY

## 实际用例：自动化 VPS 管理的几个常见场景

### 场景一：批量重启多台服务器

假设你用 Evoxt 部署了 5 台服务器跑不同的服务，某天推送了系统配置更新，需要全部重启。手动一台一台点，要登录、找到对应 VM、点重启、确认……五台下来少说十分钟。

用 API 的话，一个循环脚本，发五个 POST 请求，30 秒搞定。

python
import requests

API_KEY = "你的_API_KEY"
VM_IDS = ["vm-001", "vm-002", "vm-003", "vm-004", "vm-005"]

headers = {"Authorization": f"Bearer {API_KEY}"}

for vm_id in VM_IDS:
    resp = requests.post(
        f"https://api.evoxt.com/vm/{vm_id}/reboot",
        headers=headers
    )
    print(f"{vm_id}: {resp.status_code}")


（注：具体的 endpoint 路径以 Evoxt 官方 API 文档为准，上面只是演示逻辑结构）

Evoxt 官方维护了一个 WHMCS 插件，通过 API 实现了：自动化部署、暂停、终止、续费、备份恢复、防火墙配置、VNC 访问等功能。如果你是开发者或者小型服务商，想转售 VPS 资源，Evoxt 的 API 直接省掉了大量集成工作量。
